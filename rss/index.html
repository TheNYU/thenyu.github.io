Qu'est-ce que Nyan ?

Nyan est un moteur de jeu. Un moteur de jeu est un ensemble de fonctions qui permettent tout ce qui est nécessaire au bon fonctionnement d'un jeu vidéo. Un moteur de jeu gère entre autres les graphismes, la musique, la souris et le clavier.
Pour la série de logiciels RPG Maker — pour les versions XP, VX et VX Ace —, ce moteur de jeu s'appelle le RGSS. Nyan se propose comme une alternative au RGSS, pour le remplacer. Le but est d'améliorer à la fois les performances et les possibilités offertes. Nyan permet comme le RGSS d'utiliser le Ruby comme langage de script et a pour but de maintenir la compatibilité des scripts existants: en somme, vous pourrez normalement passer du RGSS à Nyan sans changer un seul script.
Voici un aperçu des fonctionnalités de Nyan.

De meilleures performances

Améliorations générales

Nyan propose de meilleures performances que le RGSS officiel. Cela implique un jeu qui souffre de moins de lags, qui utilise également mieux la mémoire.

Des Thread pour charger les données

Nyan généralise aussi une nouvelle manière d'améliorer les performances: les Thread. Il s'agit d'une composante déjà présente dans Ruby qui propose du multi-threading. Derrière ce terme barbare se cache un procédé assez simple: il s'agit de faire plusieurs choses à la fois. Par exemple, il sera possible d'intégrer des écrans de chargements ou de fluidifier l'expérience de jeu en préchargeant certaines données, certaines images, certaines musiques.

Une nouvelle version de Ruby

Autre source d'amélioration des performances: Nyan utilise une des dernières versions de Ruby. Tandis que RPG Maker XP, VX et VX Ace utilisaient respectivement Ruby 1.8.7, Ruby 1.9.1 et Ruby 1.9.2, Nyan fait un bond en avant en proposant le support de Ruby 2.3.2.

Nyan remplace la plupart de vos scripts

Support de toutes résolutions

Peut-être utilisiez-vous auparavant un script pour changer la résolution de votre jeu. C'est désormais fini avec Nyan qui propose d'utiliser n'importe quelle résolution et d'en changer librement en cours de jeu. Nyan est également fourni avec un système de zoom, idéal pour les jeux rétro ou pour proposer un agrandissement natif de l'écran pour les joueurs qui le trouveraient trop petit.

Mode plein écran corrigé

Le plein écran dans RPG Maker avait la fâcheuse tendance à être flou et déformé selon l'écran utilisé. Nyan corrige ceci en proposant un plein écran propre en pixel-perfect et applique des bandes noires autour du jeu de manière à éviter la déformation.

Gestion de la souris et du clavier

De nombreux jeux ont besoin de la souris mais RPG Maker ne le proposait pas. Nyan offre un support natif de la souris ce qui permet d'éviter les pertes de performances et les bugs dus à l'utilisation de tels scripts. Nyan offre également un support complet du clavier et notamment la possibilité d'entrer les noms des personnages au clavier directement.

Possibilités offertes par Nyan

Système d'interface

Nyan est livré avec un système d'interface qui lui est propre. Il propose une syntaxe DSL pour clarifier la création des interfaces et divise le nombre de lignes pour une interface par 10 en moyenne.
Un utilitaire pour la création d'interfaces sans programmation est envisagé dans le futur.

Système d'animation

Un système d'animation est également inclus. Il permet d'utiliser facilement les fonctions d'easing de Robert Penner pour dynamiser l'affichage à l'écran et proposer une expérience de jeu plus intéressante.

Module Network

Un module Network a été ajouté. Il gère la communication avec un serveur, permet d'envoyer des requêtes, de recevoir des réponses mais également de télécharger des données ou d'en envoyer. Ce module est compatible avec les serveurs PHP mais également avec les serveurs NodeJS pour envisager notamment des systèmes de combat multijoueur.

OpenGL

L'utilisation d'OpenGL est disponible avec Nyan. Cela signifie que les programmeurs pourront ajouter de la 3D à leurs jeux sans ralentissements avec l'API connue.

Multiplateforme

Nyan est multiplateforme. En effet, il peut théoriquement fonctionner sur Windows mais également sur Mac et Linux. De plus, avec un peu de connaissances en programmation, il est possible d'en faire une application Android ou iOS et de le distribuer plus largement.

Codé en Ruby et open source

Nyan est lui-même codé en Ruby. En deux mots, il est facile d'étendre ou de modifier directement le moteur pour votre projet si votre utilisation le nécessite. Le code source est disponible sur GitHub.

Côté code

Syntaxe plus efficace

Au niveau des scripts, les programmeurs trouveront une syntaxe plus efficace que la syntaxe originale du RGSS. 

@sprite = Sprite.new.dsl do
  x 10
  y 20
  bitmap = Bitmap.new("image.png").dsl do
    fill_rect Rect.new(x:0,y:20,width:10,height:30), :blue
    draw_line [0,0], [12,20], :red
  end
  opacity 140
end

Cette syntaxe permet d'utiliser à la fois la méthode dsl pour initialiser un bloc DSL et la méthode set pour utiliser les keywords arguments.

Utiliser des blocks avec le module Input

L'utilisation des blocks est généralisée au niveau des fonctions d'Input. 

Input.trigger?(:A) {  
  puts "On a appuyé sur A"
}

C'est également le cas de la plupart des fonctions de test qui proposent d'exécuter un block si true. 

# Syntaxe traditionelle
if @sprite.mouse_over?  
  puts "La souris survole le sprite"
end

# Nouvelle syntaxe
@sprite.mouse_over? {
  puts "La souris survole le sprite"
}

La précédente syntaxe reste évidemment tout à fait valable et utilisable.

Améliorations générales

De nombreuses classes ont été améliorées et ont été agrémentées de nouvelles méthodes. Vous pouvez en retrouver tout le détail dans la documentation. Il s'agit parfois d'allègements de syntaxe ou de nouvelles manières de procéder: Nyan a été conçu pour rendre le code encore plus rapide et simple à écrire.

Quand est né le projet ?

Le projet est né en 2011 dans mon esprit mais je n'avais pas les compétences pour le réaliser. J'ai ceci dit toujours modifié amplement les scripts de base de RPG Maker et finalement, il est venu Nyan en octobre 2016.

Mon projet sera-t-il compatible avec Nyan ?

Normalement, oui. Ceci dit, certains scripts spéciaux ne pourront pas fonctionner mais les fonctions de Nyan sont là pour les remplacer. Vous pouvez suivre le guide suivant pour adapter votre projet: How to migrate from RGSS to Nyan.

Mon projet RPG Maker MV est-il compatible ?

Malheureusement, non. Seuls les projets RPG Maker XP, VX et VX Ace sont compatibles.

Pourquoi n'y a-t-il pas telle ou telle fonction ?

Certaines choses sont moins prioritaires que d'autres. Par exemple, la lecture de vidéos n'est pas envisagée pour l'instant.

Peut-on utiliser Nyan sans RPG Maker ?

En effet, Nyan n'a pas besoin de RPG Maker pour fonctionner: on peut tout coder directement en Ruby.

Nyan est-il gratuit ?

Parfaitement! Et il le sera toujours.

Oui, tout à fait. Seule la mention de Nyan ainsi que mon nom — NYU — sont obligatoires. Pour le reste, la vente du jeu est parfaitement possible, sans aucune restriction ni contre-partie.

What's Nyan ?

Nyan is a game engine. A game engine is a set of functions which covers all the needed to create a video game. A game engine deals with graphics, sounds or input for instance.
The game engine for RPG Maker XP, RPG Maker VX or RPG Maker VX Ace is called RGSS. Nyan is an alternative to RGSS. Its goal is to improve performances and possibilities. Nyan is also using Ruby, as RGSS, and is compatible with most of existing scripts. In most cases, you can switch from RGSS to Nyan without changing a single line of code.
The following is an overview of Nyan's features.

Better performances

General improvements

Nyan is offering better performances than the official RGSS. Thus, the game is less laggy and makes a better use of memory.

Thread-compatible

Nyan is also using Ruby's Threads. It's a class in Ruby's core which deals with multi-threading. That means that the computer can now execute multiple blocks of code in a same time. For instance, you can now create loading screens and preloading some assets, images or sounds.

A new version for Ruby

Nyan uses one of the last versions of Ruby. While RPG Maker XP was using Ruby 1.8.7, RPG Maker VX Ruby 1.9.1 and RPG Maker VX Ace Ruby 1.9.2, Nyan uses Ruby 2.3.2 and benefits from a important performance improvement.

Nyan is better than your scripts

Multi-resolution support

Many games are using scripts to change the screen resolution. Nyan offers a simple and native alternative: you can now use the resolution you want and change it during the game. Nyan is also using a zoom system, which works really nice with retro games or for players who think that the game is too little.

Nice fullscreen

Default fullscreen in RPG Maker is often blurry and distorted following the screen used. Nyan fixes that issue with a pixel-perfect fullscreen and black rectangles on the side to prevent distortion.

Gestion de la souris et du clavier

A lot of games are using mouse. However, mouse in RGSS is often laggy and can create bugs. Nyan offers a native support for mouse to prevent fps drops and bugs. Nyan is also packed with a full keyboard support and with the possibility to enter characters' names with keyboard.

Some improvements

A full UI system

Nyan is packed with its own UI system with a DSL syntax to simplify UI creation and divides the number of lines required by ten.
A little software to create UIs without code can be created in the future.

An animation system

An animation system is also included. You can use the Robert Penner's well-known easing function to improve the visual quality of your game.

Network module

A network module was added. It deals with server exchange: it can send requests, receive responses but also download some data or send informations. This module is designed to works with PHP servers but also NodeJS servers to create for instance online battle systems.

OpenGL

OpenGL is available with Nyan. Programmers can add 3D or shaders to theirs games without lag with the class OpenGL Api.

Cross-plateform

Nyan is cross-plateform. Actually, Nyan can work on Windows but also on Mac and Linux. Moreover, with some programing basis, you can port your game to Android or iOS and share it on mobile devices too.

100% Ruby & open source

Nyan is coded in Ruby: it's more convenient to extend or modify the engine if you need to. The source code is available on GitHub.

Code-side

More concise and efficient syntax

Programmers will find a syntax more efficient than the RGSS one. 

@sprite = Sprite.new.dsl do
  x 10
  y 20
  bitmap = Bitmap.new("image.png").dsl do
    fill_rect Rect.new(x:0,y:20,width:10,height:30), :blue
    draw_line [0,0], [12,20], :red
  end
  opacity 140
end

You can use the dsl method to initialize a DSL block and the set method to use keywords arguments.

Pass blocks to Input methods

You can now pass blocks to Input-related methods. 

Input.trigger?(:A) {  
  puts "A triggered!"
}

It's also possible for most of the methods returning a boolean: the block is executed if true. 

# Default syntax
if @sprite.mouse_over?  
  puts "The mouse is over the sprite"
end

# New syntax
@sprite.mouse_over? {
  puts "The mouse is over the sprite"
}

The old syntax is however correct ans usable.

General improvements

A lot of class were improved and new methods were added. You can see all the changes in the documentation. Sometimes, it's a improvement on the syntax or a new way to achieve something. Nyan was designed to improve efficiency and productivity. Code is more concise and simple.
